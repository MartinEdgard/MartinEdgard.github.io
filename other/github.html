Våre tanker om Github

Å bruke github gikk bra uten noen strore problemer. Vi fikk alle testet at det gikk ann å pushe/pulle for alle. Vi brukte litt tid på å bli vant til denne typen arbeidsflyt.

Vi ble aldri veldig kjent med github og det tok ikke lang tid før vi fant ut at vi mye heller ville jobbe på prosjektet i liveshare.

GitHub viste seg å være svært nyttig da vi støtte på problemer med nettsiden. Ved å bruke GitHub kunne vi enkelt spore endringene vi hadde gjort, noe som førte oss til å finne årsaken til feilen.

Våre tanker om liveshare

Vi fant raskt ut at det finnes en extension som heter liveshare. Denne har vi benyttet oss av under hvert eneste gruppemøte. Siden vi har jobbet synkront på skolen opptil flere ganger i uken, har vi ikke måtte pushe/pulle for å dele koden mellom oss. Vi brukte litt tid på å finne ut av at man kunne dele live server slik at alle kunne se nettsiden på sin egen skjerm. Det er noen ulemper ved å bruke live share. En av de er at noen ganger blir du festet til en annen sin musepeker uten en mulighet for å komme ut av det. Noe som irriterte oss veldig er når flere koder samtidig og en person benytter seg av funksjonen Ctrl + Z, fjerner dette andre sitt arbeid i tillegg.
