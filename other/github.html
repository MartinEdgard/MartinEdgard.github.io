GitHub

Våre tanker om GitHub

Å bruke GitHub i dette prosjektet gikk generelt veldig bra, og vi støtte ikke på noen store utfordringer. Alle i gruppa fikk testet og bekreftet at vi både kunne pushe og pulle endringer.

GitHub viste seg å være spesielt nyttig når vi møtte på problemer med nettsiden vår. Når vi støtte på feil, kunne vi bruke GitHub til å spore endringene vi hadde gjort i koden på en lett måte. Ved å se gjennom historikken kunne vi identifisere når og hvor feilen ble laget. Dette gjorde at vi forstå viktigheten av hyppige push commits med gode meldinger.

Selv om vi brukte LiveShare for selve utviklingsprosessen, ga GitHub oss erfaring med å bruke et verktøy som kommer til å bli veldig nyttig for oss i fremtidige samarbeidsprosjekter.

Våre tanker om LiveShare

Vi oppdaget raskt at det finnes en utvidelse som heter LiveShare, og denne har vært et nyttig verktøy under alle våre gruppemøter. Siden vi har jobbet synkront på skolen flere ganger i uken, har vi ikke hatt behov for å pushe eller pulle kode mellom oss – noe som har forenklet samarbeidet betraktelig. Etter litt eksperimentering fant vi også ut at det er mulig å dele en live server gjennom Live Share, slik at alle på teamet kunne se nettsiden på sine egne skjermer. Da ungikk vi å måtte alle sitte på samme side av bordet for å kunne se hvordan siden så ut.

Likevel har vi støtt på noen utfordringer ved bruk av Live Share. Et problem er at man av og til blir ufrivillig «festet» til en annen persons musepeker og dermed bli dratt rundt på siden etter en annen person. Dette var spesielt irriterende hvis to personer prøvde å kode samtidig, og den ene personen ble "teleportert" til den andre personen mens begge to skriver. En annen utfordring, som virkelig irriterte oss, oppstod når flere skrev kode samtidig: hvis en person brukte Ctrl + Z for å angre endringer, kunne det fjerne kode som andre nylig hadde skrevet.

Til tross for disse ulempene har Live Share vært et svært nyttig verktøy som har gjort samarbeid enklere og mer effektivt.

Positive tanker om GitHub og LiveShare

GitHub

  • Gjør konflikthåndtering ekstremt enkelt
  • Gir alle gruppemedlemmene muligheter til å se hvilke nedringer som ble gjort siden forrige gang de var i dokumentet
  • Muligheter for å finne tidligere kode

LiveShare

  • Når man er i læringsfasen av HTML og CSS, var det enklere å lære sammen ved å kunne samarbeide i "real time"
  • Man kan enkelt forklare eller "følge" andre gruppemedlemmer da man kan velge å pinne seg selv til dem
  • Slipper eventuelle konflikter i koden som kunne oppstått hvis flere pushet kode samtidig
